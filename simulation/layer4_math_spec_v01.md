## 核心选择：代数合成 vs 动力学耦合

### 选择：代数合成

**理由**：
1. **单 γ 体系天然稳定**：交叉项 k×min(Φ_g, Φ_e) 是代数合成，不参与步进计算
2. **合约结构不变**：保持单 γ 设计，合约简洁
3. **物理直觉正确**：耦合项只在步进后组合评分，不影响动力学

### 动力学耦合的问题

动力学耦合需要：
```
dΦ_g/dt = ..., dΦ_e/dt = ...
```

但原始 PDE 规范是**事件驱动的离散更新**，不是连续的微分方程。动力学耦合需要改写原始规范，风险高。

## 数学规范

### 1. 原始 PDE 规范（基于 Mesa 模拟器）

**势位定义**：
- **类型**：连续势位（用于代数合成）
- **表示**：连续变量 Φ
- **来源**：Shi-Graph 拓扑传播 + 使用事件

**衰减机制**：
- **类型**：单 γ 体系
- **公式**：Φ' = Φ + gain - γ × Φ
- **参数**：γ = 0.2

**更新频率**：每7天更新一次

### 2. 耦合框架定义（代数合成）

**核心思想**：耦合项是代数合成，不是动力学耦合

**势位定义**：
- **图势位 Φ_g**：来自 Shi-Graph 拓扑传播
- **事件势位 Φ_e**：来自使用事件
- **总势位**：`Φ_total = Φ_g + Φ_e + k×min(Φ_g, Φ_e)`

**更新规则（代数合成）**：

```
步进 1：计算 Φ_g 和 Φ_e
  Φ_g' = Φ_g + gain_g - γ × Φ_g
  Φ_e' = Φ_e + gain_e - γ × Φ_e

步进 2：代数合成（耦合项不参与步进）
  Φ_total = Φ_g' + Φ_e' + k×min(Φ_g', Φ_e')
```

**关键点**：
- Φ_g 和 Φ_e 分别受 γ 衰减
- 耦合项 k×min(Φ_g, Φ_e) 只在步进后组合
- 耦合项不参与步进计算

### 3. 衰减机制

**单 γ 体系**：
```
Φ_g' = Φ_g + gain_g - γ × Φ_g
Φ_e' = Φ_e + gain_e - γ × Φ_e
Φ_total' = Φ_g' + Φ_e' + k×min(Φ_g', Φ_e')
```

**衰减参数**：
- γ = 0.2（默认值）
- 更新频率：每7天

## 稳定性分析

### 4. 稳定性条件

**定理**：在单 γ 体系下，耦合框架是稳定的（k 无上界）。

**证明**：

**步进 1**：Φ_g 和 Φ_e 分别更新
```
Φ_g' = (1-γ) × Φ_g + gain_g
Φ_e' = (1-γ) × Φ_e + gain_e
```

**步进 2**：代数合成
```
Φ_total' = Φ_g' + Φ_e' + k×min(Φ_g', Φ_e')
```

**稳定性分析**：

步进 1 的稳定性：
```
|Φ_g' - Φ_g*| = |(1-γ) × (Φ_g - Φ_g*)|
|Φ_e' - Φ_e*| = |(1-γ) × (Φ_e - Φ_e*)|
```

当 0 < γ < 1 时，|1-γ| < 1，因此 Φ_g 和 Φ_e 指数收敛。

步进 2 的稳定性：
- 耦合项 k×min(Φ_g, Φ_e) 是**代数运算**，不是动力学更新
- 它只是组合 Φ_g 和 Φ_e，不改变收敛性

**结论**：
- k **没有稳定性上界**（因为不参与步进）
- k 可以取任意正值，只要合约能接受
- 稳定性只取决于 γ ∈ (0, 1)

### 5. 与 EigenTrust 的关系

**声明**：本框架**受 EigenTrust 启发**，不是同构。

**相似性**：
- 都是单衰减系数（γ vs α）
- 都是迭代收敛
- 都有全局稳定性保证

**差异**（不是同构）：
- EigenTrust 是**全局平均**，本框架是**局部增益**
- EigenTrust 是**马尔可夫链**，本框架是**确定迭代**
- EigenTrust 有**随机性**，本框架没有

**严格声明**：
- "受 EigenTrust 启发" ✅
- "与 EigenTrust 同构" ❌（数学本质不同）

## ZK 电路可行性

### 6. ZK 约束分析

**耦合项计算**：
```
输入：事件列表 E
输出：Φ_total = Φ_g + Φ_e + k×min(Φ_g, Φ_e)
```

**ZK 约束分解**：

| 操作 | 约束数 | 说明 |
|------|--------|------|
| 事件过滤 | ~N×10 | 每个事件需要 10 个约束 |
| min() 比较器 | ~200 | 比较两个数的大小 |
| 定点数乘法 | ~400 | k × min() |
| 加法 | ~3 | Φ_g + Φ_e + cross |
| **总计** | **~N×10 + 600** | |

**注**：N 是事件数量。

**可行性**：
- ✅ 可行：约束数与事件数量成正比
- ✅ 可扩展：可以处理大规模事件
- ⚠️ 注意：N×10 + 600 约束需要优化

## 结论

### 8. 关键发现

1. **k 无上界**
   - 耦合项是代数合成，不参与步进
   - 稳定性只取决于 γ ∈ (0, 1)
   - k=0.05 绝对安全

2. **单 γ 体系正确**
   - 与合约实现一致
   - 物理直觉正确
   - 稳定性有保证

3. **代数合成选择正确**
   - 不改变原始 PDE 规范
   - 不增加复杂度
   - 与现有代码兼容

4. **EigenTrust 是启发，不是同构**
   - 降级声明为"受启发"
   - 不声称数学同构

### 9. 建议

1. **保持单 γ 设计**
2. **k=0.05 安全**（k 无上界）
3. **保持代数合成**
4. **等待 ZK 电路详细评估**
