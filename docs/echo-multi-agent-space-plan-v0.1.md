## 通信协议（今天痛点的解决方案）

### 问题 1：Talis 不回应
**原因**：飞书群里 Agent 之间不能自动触发（树形可见性限制）。
**解决**：
- 短期：每次讨论前 **Founder 手动 @Talus**（蓝色 @）激活
- 中期：Lucifer 配置 Talus 自动监听群消息（需要对方配合）
- 长期：ECHO 协议内置「Agent 唤醒机制」

### 问题 2：文档太多太快
**原因**：Agent 7×24 工作，人类需要休息。
**解决**：
- **节奏控制**：每天最多 1 份核心文档，给人类 24h review 时间
- **分级产出**：
  - P0（紧急）：当天发，当天审（如设计漏洞修复）
  - P1（常规）：今天发，明天审（如新功能设计）
  - P2（储备）：本周发，本周审（如技术调研）

### 问题 3：决策权不清
**原因**：Agent 提出建议，但不知道谁拍板。
**解决**：
- **Seaman 拥有最终决策权**（技术架构）
- **雅婷拥有产品决策权**（方向和节奏）
- **Agent 只执行和整理，不做最终决策**
- 所有关键决策需 Seaman 或雅婷确认

## 协作流程

```
需求提出（雅婷/Seaman/人类）
    ↓
Agent 认领（雨娃协调分配）
    ↓
并行工作（各 Agent 在自己领域产出）
    ↓
雨娃整合（汇总为统一文档）
    ↓
人类 review（24h 窗口）
    ↓
修改/确认
    ↓
部署上线
    ↓
进入下一轮
```

**示例**：
- Talus 发现漏洞 → 雨娃整理成修正建议 → Seaman review → 雅婷确认 → 部署
