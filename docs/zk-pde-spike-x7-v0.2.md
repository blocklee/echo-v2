## 二、关键发现：数学模型不匹配

### 模拟器实际动态

| Round | Mean φ | Max φ | Min φ | 每步增长率 |
|-------|--------|-------|-------|-----------|
| 0 | 1.00 | 1.00 | 1.00 | — |
| 10 | 6.17 | 54.49 | 1.17 | 1.49x |
| 50 | 25.57 | 112.90 | 12.78 | 1.10x |
| 100 | 51.62 | 158.96 | 32.38 | 1.05x |
| 150 | 80.57 | 256.47 | 55.87 | 1.04x |
| 200 | 101.54 | 206.74 | 75.47 | 1.03x |

### 我的 PDE 模型（v0.3）

```
φ_{t+1} = φ_t + dt·(D·k(φ)·∇²φ_t - γ·φ_t + S_base)
```

这是**加性**（additive）更新：每步增量 = D·扩散 + S - γ·衰减

预测第一 step：φ_1 ≈ 1.0 + 0.35 = **1.35**（φ_0=1.0，dt=1，S_base=0.4）

### 实际模拟器

观察到第一 step 效果：**φ_10/φ_0 max = 54.49**（10步内）

即：**乘性**（multiplicative）反馈，远超加性 PDE 预测。

### 根本性不匹配

| | 我的 PDE 模型 | 模拟器实际 |
|--|--|--|
| 类型 | 加性 Euler | 乘性反馈 |
| 每步增长率 | ~0.35（线性） | 1.03-1.49x（指数） |
| 物理含义 | 扩散+源-衰减 | 富者更富（图拓扑强化） |

### 影响

1. **模拟器就是势位场的真实动态**：模拟器反映的是 ECHO 创作生态中"优势积累"的物理过程，这不是 bug，是 feature
2. **我的 PDE 框架是错的**：连续 PDE 假设加性演化，但 ECHO 实际机制是乘性反馈
3. **连续链理论需修正**：云子论文的 PDE 方程形式需要检查——如果实际离散化是乘性的，积分路径验证的含义会不同

## 三、ZK 电路设计影响

### 乘性 vs 加性的 ZK 约束差异

**加性更新（我的旧模型）：**
```
constraint: phi_next - phi_current - dt*(D*L@phi + S - gamma*phi) = 0
```
→ 二次约束，可直接用 PLONK 门

**乘性更新（模拟器实际）：**
```
constraint: phi_next = phi_current * (1 + feedback_factor)
```
→ 需要倒数约束或比值约束（a * b^{-1} = c），PLONK 可支持但门数增加

### 推荐的 ZK 约束方案

**方案 C（可行）：界乘法约束**
```
证明：phi_t * (1 + g_min) >= phi_{t+1} >= phi_t * (1 + g_max)
其中 g_min, g_max 由图拓扑决定（可预计算）
```
→ 只需比较约束，不需要精确乘法/倒数

**方案 B（更精确）：比值约束**
```
PLONK 支持 a * b^{-1} = c 类约束
证明：phi_{t+1} * phi_t^{-1} = growth_factor
```
→ 需要字段倒数（有限域运算），可行

### 定点数 Q32.32 溢出风险

- φ 范围：round 200 时 [75.5, 206.7]
- Q32.32 上限：2^32 ≈ 43亿，远大于 206.7
- **结论：定点数范围充足，无溢出风险** ✓

## 四、约束规模重估（基于乘性动态）

| 操作 | 约束类型 | 数量 |
|------|---------|------|
| φ_t * inv(φ_t) = 1（自倒数） | 1 constraint | O(n) |
| growth_ratio 界约束 | 2 constraints/节点 | O(n) |
| 图拓扑反馈求和 | O(m) | O(m) |
| **单步总计** | | **O(n+m)** ≈ 11,000（与v0.1一致）|

结论：约束规模不变，但**约束类型变了**（乘法/倒数替代加法）

## 六、v0.2 结论

**PDE 数学链路存在根本性模型不匹配：**
- 我的 Euler 加性 PDE 预测线性增长（~0.35/step）
- 实际模拟器显示指数增长（1.03-1.49x/step）
- **ECHO 势位场实际机制是乘性反馈，不是加性扩散**

**ZK 电路设计调整：**
- 从"证明加性 PDE 一步"改为"证明乘性增长在界内"
- 约束规模不变，约束类型需改为比值/界约束
- 这反而更符合 ECHO 创作生态的本质（复利效应）
