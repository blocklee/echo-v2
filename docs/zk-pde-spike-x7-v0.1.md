## 一、核心 PDE（来自 v0.3 框架）

ECHO 势位演化采用 Euler 前向离散：

```
Φ_{t+1} = Φ_t + Δt · [D·k(Φ_t)·∇²Φ_t - γ·Φ_t + S_t]
```

其中：
- **Φ** = 势位（potential），向量形式
- **D** = 基础扩散系数（scalar）
- **k(Φ)** = 动态扩散系数 = 1.2 + 0.3·tanh(beta·(Φ-0.5))，beta 控制陡度
- **γ** = 衰减率（scalar）
- **S_t** = 源项 = ξ/5.0，ξ = (D+A+K+P)/4（四权等权平均）
- **∇²** = 图拉普拉斯算子（graph Laplacian），矩阵形式 L

## 二、有限差分离散化

### 2.1 空间离散化（图拉普拉斯）

图拉普拉斯定义为：
```
L = I - D^{-1}·A
```
或归一化版本：
```
L = I - D^{-1/2}·A·D^{-1/2}
```

其中 D 是度对角矩阵，A 是邻接矩阵。

对 n 个节点的系统，L 是 n×n 矩阵，∇²Φ = L @ Φ 是矩阵-向量乘法。

### 2.2 时间离散化（Euler 前向）

```
Φ_{t+1} = Φ_t + dt · (D·k(Φ_t)·(L@Φ_t) - γ·Φ_t + S_t)
```

展开为分量形式（对第 i 个节点）：
```
Φ_i^{t+1} = Φ_i^t + dt · [D·k(Φ_i^t)·Σ_j L_{ij}·Φ_j^t - γ·Φ_i^t + S_i^t]
```

**关键观察：** 每一步迭代需要：
1. 计算 L @ Φ（n×n 矩阵与 n维向量乘法，O(n²) 或优化后 O(n)）
2. 逐节点计算 k(Φ_i)（涉及 tanh）
3. 加权求和得到下一步 Φ

## 三、ZK 友好的离散化形式

### 3.1 非线性项的处理

**问题：** tanh 是非线性函数，PLONK 约束只支持有限域上的二次约束。

**方案 1：查找表（LUT）**
- 将 tanh 在 [0,1] 区间预计算为定点数表
- 在 ZK 电路中用 lookup constraint 替代直接计算
- PLONK 支持 fixed-base table lookup

**方案 2：Chebyshev 多项式逼近**
- tanh(x) ≈ Σ c_n·T_n(x)，截断到固定阶数
- PLONK 约束可组合任意阶乘法门
- 精度与阶数权衡：阶数越高精度越高，但约束数增加

**推荐：** 方案 1（LUT），约束数可控，适合 ξ 档位有限（0-5）场景。

### 3.2 矩阵-向量乘法的 ZK 映射

**问题：** L @ Φ 涉及 n×n 矩阵与 n维向量乘法。

**方案：**
- 将 L 视为常量（在 ECHO 图结构固定时，L 是公开常数）
- 每个节点 i 的计算：Σ_j L_{ij} · Φ_j
- 这是一个 **注意力机制的简化形式**（类似于二次约束）

**约束映射：**
- 拉普拉斯系数 L_{ij} 作为 public input 或 constant
- Φ_j 作为 witness（私密输入）
- 每对 (i,j) 贡献一个乘法门

**约束数估算：**
- n = 节点数（假设 ECHO 活跃节点规模待定）
- 每个 L_{ij} · Φ_j 乘法 → O(n²) 约束
- 实际可优化：L 是稀疏矩阵（节点只与少数邻居连接）

### 3.3 定点数精度（Q32.32 接口）

**问题：** 合约层使用 uint256 存储 Q32.32 定点数，ZK 电路需保持精度一致。

**精度分析：**
- Q32.32：32 位整数部分 + 32 位小数部分
- Φ 范围：[0, 1]（势位归一化）
- D·k(Φ)·∇²Φ 各项量级估算：
  - D：标度系数，~0.1（需实验确认）
  - k(Φ)：范围 [1.2, 1.5]
  - ∇²Φ：图拉普拉斯作用，结果范围与 Φ 相同
- 乘法中间结果：Q32.32 × Q32.32 = Q64.64，需截断到 Q32.32

**截断误差上界：**
```
|error| ≤ 2^{-32} ≈ 2.3×10^{-10}
```
对 ECHO 势位演化精度足够。

## 四、约束规模估算

### 4.1 单步迭代约束数（估算）

| 操作 | 约束类型 | 数量级 |
|------|---------|--------|
| k(Φ_i) 查找（tanh LUT） | 1 constraint/节点 | O(n) |
| D·k(Φ_i) 乘法 | 1 constraint/节点 | O(n) |
| L @ Φ（矩阵向量乘） | 2 constraints/边 | O(m)，m=边数 |
| 加权求和 | 1 constraint/节点 | O(n) |
| dt·(...) 标量乘法 | 1 constraint/节点 | O(n) |
| Φ_i + dt·...（加法） | 1 constraint/节点 | O(n) |
| **总计/步** | | **O(n+m)** |

### 4.2 总体约束规模（待实际数据）

**关键未知：**
1. ECHO 网络规模 n（活跃节点数）—— 影响约束总数
2. 图结构稀疏度 m/n（平均度数）—— 影响矩阵乘法约束
3. 时间步数 T（需验证的时间窗口）—— 影响总约束数

**假设（待验证）：**
- n = 1000（保守估计）
- m/n = 20（平均度 20）
- T = 10（滚动验证窗口）

**估算：**
- 单步约束 ≈ O(n+m) ≈ O(n·(1+m/n)) ≈ 1000×21 = 21,000 constraints
- T=10 步总约束 ≈ 210,000 constraints

**对比参考：** PLONK 支持 ~10⁶ 级约束，ZKSNARK 验证单电路上限约 10⁸ 约束。

## 五、链上/链下分层建议

### 5.1 分层方案

```
链下（ZK 证明者）：
  - 运行 PDE 迭代 T 步
  - 生成 ZK 证明（PLONK/GPARIS）
  - 提交：公共输入（Φ_0, S_t 序列）+ ZK 证明

链上（验证者）：
  - 验证 ZK 证明
  - 检查公共输入一致性
  - 更新合约状态（Φ_T）
```

### 5.2 信任模型

- **优点：** 链上只验证证明，不重跑 PDE；Gas 成本大幅降低
- **风险：** ZK 电路正确性依赖电路约束与 PDE 语义等价性
- **缓解：** 需要独立团队对电路约束做形式化验证

## 八、结论

PDE → ZK 链路数学上可行，核心挑战是：

1. **非线性项处理**（tanh → LUT 方案可行）
2. **矩阵乘法约束规模**（稀疏性决定成败）
3. **精度对齐**（Q32.32 与连续 PDE 的误差上界需量化）
