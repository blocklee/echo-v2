🎯 选一种方式

⭐ 推荐 · 最简单

方式一：Docker 部署（一条命令）

适合：有 Docker 的环境（云服务器、本地电脑）

优点：一条命令搞定所有依赖，环境隔离，不污染系统

# 1. 进入模拟器目录 cd echo-v2/simulation # 2. 构建镜像（首次约2-3分钟） docker build -t echo-simulator . # 3. 启动容器 docker run -d -p 8000:8000 --name echo-api echo-simulator # 4. 检查状态 docker logs echo-api
✅ 完成后访问：http://你的服务器IP:8000/health
📖 API 文档：http://你的服务器IP:8000/docs

常用命令：

# 停止 docker stop echo-api # 重启 docker start echo-api # 删除容器 docker rm -f echo-api # 查看日志 docker logs -f echo-api
方式二：Python 直接部署

Python 直接运行

适合：不想用 Docker，服务器已有 Python 3.8+

# 1. 进入目录 cd echo-v2/simulation # 2. 安装依赖（首次） pip3 install -r requirements.txt # 3. 启动（或运行 start.sh） python3 echo_simulator_api_v2.py
依赖清单：fastapi, uvicorn, pydantic, mesa, networkx, scipy, numpy
如果安装慢，可以用国内镜像：pip3 install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
方式三：一键脚本（最懒）

运行 start.sh

适合：Linux/Mac 环境，想偷懒

# 给脚本执行权限（首次） chmod +x start.sh # 运行 ./start.sh
脚本会自动检查 Python、安装依赖、启动服务，全程无脑。

🔧 云服务器部署注意事项

安全组/防火墙放行 8000 端口

大部分云服务器默认会阻止外部访问 8000 端口，需要手动放行。

阿里云/腾讯云/AWS/华为云：

  • 登录云厂商控制台
  • 找到「安全组」或「防火墙」
  • 添加规则：入方向，TCP 8000 端口，来源 0.0.0.0/0

Linux 系统防火墙（ufw/iptables）：

# Ubuntu/Debian（ufw） sudo ufw allow 8000/tcp # CentOS（firewalld） sudo firewall-cmd --permanent --add-port=8000/tcp sudo firewall-cmd --reload
⚠️ 关键点：即使代码运行正常，如果安全组没放行，外部也访问不了。
雨娃之前遇到的问题就是这个——VPS 本地运行正常，但外部连接超时。

📋 常见问题

Q: pip install 很慢？
A: 换国内镜像：pip3 install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

Q: 端口被占用？
A: 修改 echo_simulator_api_v2.py 最后一行的 port=8000 改成别的，比如 port=8080

Q: mesa 安装失败？
A: Mesa 需要较新 Python（3.9+），如果系统 Python 太旧，用 Docker 方式最稳。

Q: 启动后无法从外部访问？
A: 99% 是安全组/防火墙问题。检查云厂商控制台的安全组规则。

Q: 内存不足？
A: 大规模模拟（100+ 用户）需要较多内存。小实验（6 创作者 × 20 用户）1GB 内存足够。
