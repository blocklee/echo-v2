📡 API 小白教程

3分钟上手 · 不需要写代码 · 复制粘贴就行
什么是 API？
想象 API 是一个自动售货机。你按按钮（发送请求），它给你饮料（返回数据）。你不需要知道机器里面怎么运作，只需要知道按哪个按钮、给什么参数。

🎯 今天你能做什么

  • ✅ 查看后端是否活着（健康检查）
  • ✅ 查看系统配置（创作者类型、用户角色）
  • ✅ 运行一次模拟实验（30天生态演化）
  • ✅ 对比两种策略（锦标赛）
  • ✅ 对比 ECHO vs 传统平台（A/B实验）

🛠️ 准备工作

1 打开终端（命令行）
Windows：按 Win + R，输入 cmd，回车
Mac：按 Command + 空格，输入 Terminal，回车
Linux：你肯定已经会了
2 安装 curl（一次性）

cURL 是一个命令行工具，用来和 API 对话。大多数电脑已经自带。

curl --version

如果看到版本号（比如 curl 8.0），说明已安装。如果提示"找不到命令"：

  • Windows：去 curl.se/windows 下载，解压到桌面
  • Mac：已经自带，不需要装

🚀 开始实验

实验 1：检查后端是否活着

复制下面这行命令，粘贴到终端，回车：

curl http://localhost:8000/health

如果后端在运行，你会看到：

{"status":"ok","version":"4.0.0","engine":"Mesa v4.0","capabilities":["single_run","tournament","ab_test","robustness"]}
✅ 看到 "ok" 就说明后端活着！
❌ 如果很久没反应或报错，说明后端没启动。找雨娃或技术小伙伴帮忙启动。

实验 2：查看系统配置

curl http://localhost:8000/config

你会看到创作者类型、用户角色、四权力定义等配置。这个接口告诉你可以怎么配置实验参数。

实验 3：运行一次模拟实验（核心！）

这是最重要的接口。发送一个"实验请求"，后端会模拟 30 天的生态演化，然后给你完整结果。

curl -X POST http://localhost:8000/run \ -H "Content-Type: application/json" \ -d '{"simulation_config":{"duration_days":10,"num_creators":4,"num_users":10},"environment_params":{"platform_fee_percent":5}}'
参数解释：
duration_days: 10 — 模拟 10 天（可以改 30/60/90）
num_creators: 4 — 4 个创作者（可以改 6/10/100）
num_users: 10 — 10 个用户（可以改 20/50/500）
platform_fee_percent: 5 — 平台抽成 5%（可以改 0/10/30）

等待 5-10 秒，你会看到一大段 JSON 数据。重点看这些字段：

{ "run_id": "run_20260518_...", "summary": { "total_events": 245, ← 总共发生了多少事件 "total_revenue": 156.80, ← 创作者总收入 "creator_satisfaction": 6.2, ← 创作者平均满意度（0-10） "ecosystem_health": 0.35 ← 生态健康度（0-1） }, "timeseries": [...], ← 每天的详细数据 "creator_results": [...], ← 每个创作者的结果 "asset_lifecycle": [...], ← 每个资产的完整轨迹 "s_graph_data": {...} ← S-Graph 图结构 }
💡 看不懂 JSON？没关系！把返回结果复制到 jsonformatter.org，它会帮你格式化得漂漂亮亮。

实验 4：策略锦标赛（对比两种策略）

想知道"保守型"创作者和"开放型"创作者谁赚得多？

curl -X POST http://localhost:8000/run/tournament \ -H "Content-Type: application/json" \ -d '{"simulation_config":{"duration_days":10,"num_creators":4,"num_users":10},"strategies":["保守型","开放型"],"n_rounds":5}'
strategies — 要对比的策略（可以改 ["保守型","开放型","平衡型","策略型"]）
n_rounds: 5 — 跑 5 轮取平均（可以改 10/20/50，越多越准但越慢）

返回结果里重点看：

  • ranking — 策略排名（谁第一谁第二）
  • statistical_significance — 统计显著性（p < 0.05 说明差异真实存在）
  • effect_size — 效应量（差异有多大）

实验 5：A/B 实验（ECHO vs 传统平台）

对比 ECHO 生态和传统平台（高抽成 + 无四权力），看谁更好。

curl -X POST http://localhost:8000/run/ab \ -H "Content-Type: application/json" \ -d '{"simulation_config":{"duration_days":10,"num_creators":4,"num_users":10},"n_rounds":5}'

重点看：

  • echo_group.mean_revenue vs control_group.mean_revenue — 收入对比
  • comparison.hypothesis_test_revenue.p_value — p 值（<0.05 说明 ECHO 真的更好）
  • comparison.hypothesis_test_revenue.effect_interpretation — 效应解释（小/中/大）

📊 结果怎么看

p 值（统计显著性）

  • p < 0.05 ✅ 差异真实存在，不是偶然
  • p >= 0.05 ❌ 差异可能是偶然，需要更多数据

Cohen's d（效应量）

  • < 0.2 — 可忽略（几乎没差别）
  • 0.2 ~ 0.5 — 小效应（有差别但不大）
  • 0.5 ~ 0.8 — 中效应（明显差别）
  • > 0.8 — 大效应（巨大差别）

置信区间（95% CI）

比如 [108, 130] 表示：有 95% 的把握真实平均值在这个范围里。区间越窄越精确。

🎮 进阶玩法

改参数看效果

比如你想测试"平台抽成 30% 会发生什么"：

curl -X POST http://localhost:8000/run \ -H "Content-Type: application/json" \ -d '{"simulation_config":{"duration_days":10,"num_creators":4,"num_users":10},"environment_params":{"platform_fee_percent":30}}'

对比 platform_fee_percent: 5 和 30 的结果，就能看出高抽成对生态的影响。

改用户数量

# 小生态 "num_creators": 6, "num_users": 20 # 大生态 "num_creators": 50, "num_users": 200
注意：用户越多，运行时间越长。50 创作者 × 200 用户 × 30 天可能需要 1-2 分钟。

🆘 常见问题

Q: 命令太长，容易打错？
A: 用文本编辑器（记事本/VS Code）写好命令，复制粘贴到终端。不要手打。

Q: 返回结果太乱，看不懂？
A: 复制到 jsonformatter.org 格式化。

Q: 后端连不上？
A: 确认后端已启动（curl http://localhost:8000/health 返回 ok）。如果还是不行，可能是端口被占或防火墙阻挡，找技术小伙伴。

Q: 模拟运行太慢？
A: 减少天数或用户数量。比如 duration_days: 5, num_users: 5 会快很多。

Q: 想做更复杂的实验？
A: 看完整 API 文档：api-v4-documentation.html
