## 核心架构

### ECHO 四环节
1. **Evidence** - 确权证据与权利声明
2. **Contract** - 合约控制
3. **Hash/History** - 哈希留痕与历史记录
4. **Operation/Obligation** - 操作审计与义务反馈

### 五层结构
1. 主体身份层
2. 数据标识层
3. 权利表达层
4. 合约执行层
5. 审计反馈层

### 六类合约
1. 数据登记合约
2. 权利声明合约
3. 授权控制合约
4. 用途控制合约
5. 行为日志合约
6. 责任处置合约

## 三类场景
- 医疗公共数据
- 轨道交通数据
- 社区治理数据

## 待审阅要点
1. ECHO 协议定义是否准确
2. 合约结构是否可行
3. 场景适配是否合理
4. 与当前 ECHO 设计有无冲突
