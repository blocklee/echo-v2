# ECHO 私权利四权 → 城市公共数据治理映射方案

## 一、核心映射框架

原 ECHO 协议以**私权利四权**为核心架构：
- **使用权**：对数据/数字资产进行访问、调用、加工的权利
- **衍生权**：基于原始数据进行再加工、产生新成果的权利
- **扩展权**：将数据或成果对外传播、共享、经营的权利
- **收益权**：从数据使用、加工、经营中获得经济回报的权利

面向城市公共数据治理场景，本文将四权映射为**"访问加工—衍生创新—共享经营—收益审计"四环节治理闭环**，在保留原协议"确权—授权—使用—追责"核心逻辑的基础上，适配公共数据的受托管理属性、行政监管框架和公共利益导向。

---

## 二、四权映射详解

### （一）使用权 → 访问加工环节

**原协议内涵**：使用权是私权利的基础层，指权利主体在授权范围内对数据资产进行访问、查询、调用和加工的权利。使用权的边界由授权合约界定，超出授权范围的使用构成越权。

**城市公共数据适配**：
- **访问权**：主体在授权条件下查看、查询公共数据的权利，受身份认证、权限白名单和最小必要原则约束
- **调用权**：通过标准接口获取数据服务的权利，受调用频率、数据粒度和输出形式限制
- **加工权**：在沙箱或隐私计算环境中对数据进行清洗、标注、计算和分析的权利，原始数据原则上不出域

**合约模块**：授权控制合约 + 用途控制合约
- 授权控制合约：核验主体资质、授权范围、有效期限
- 用途控制合约：校验用途声明与授权目的是否匹配，拒绝越权调用

**行政监管映射**：使用权争议由"合约自动校验—主管部门复核—行政复议"三级机制处理。智能合约承担第一层自动校验，数据主管部门承担第二层行政裁定，行政复议保留为第三层权利救济。

---

### （二）衍生权 → 衍生创新环节

**原协议内涵**：衍生权是私权利的创造层，指权利主体基于已有数据资产进行再创作、再加工，产生新数据产品、新模型或新知识的权利。衍生成果的权利归属由原始授权合约和衍生合约共同界定。

**城市公共数据适配**：
- **加工衍生权**：在合规条件下对公共数据进行统计分析、模型训练、算法开发的权利
- **模型训练权**：使用脱敏或聚合数据训练人工智能模型的权利，受伦理审查和算法备案约束
- **成果归属权**：对衍生成果（如预测模型、分析报告、数据产品）主张权利的权利，但需回溯原始数据来源

**合约模块**：权利声明合约 + 数据登记合约（衍生登记）
- 权利声明合约：记录衍生主体、原始数据单元ID、加工方式、成果形式
- 数据登记合约：为衍生成果生成新的数据单元ID，建立与原始数据的血缘关系

**行政监管映射**：衍生权争议涉及知识产权部门裁定和伦理委员会审查。医疗场景的科研衍生需通过伦理审批；交通场景的模型训练需通过算法备案；社区场景的服务创新需通过业务主管部门评估。

---

### （三）扩展权 → 共享经营环节

**原协议内涵**：扩展权是私权利的流通层，指权利主体将数据资产或衍生成果对外提供、共享、交易或经营的权利。扩展权的行使通常需要更高层级的授权，并受收益分配规则约束。

**城市公共数据适配**：
- **共享权**：在授权范围内向第三方提供数据或数据服务的权利，受共享范围、共享目的和接收方资质限制
- **经营权**：将数据产品投入市场流通、提供数据接口服务的权利，但公共数据经营需以公共价值回馈为前提
- **接口服务权**：通过API向外部系统提供数据查询、验证或计算服务的权利

**合约模块**：授权控制合约（扩展授权）+ 责任处置合约
- 授权控制合约：扩展授权需单独申请，明确共享范围、经营期限、收益规则
- 责任处置合约：越权共享、超范围经营触发暂停授权、通知监管、扣减信用评分

**行政监管映射**：扩展权行使需经数据主管部门审批。涉及跨部门共享的，由数据主管部门协调；涉及政企合作的，需签订公共数据授权运营协议；涉及市场化经营的，需明确公共收益回馈比例。

---

### （四）收益权 → 收益审计环节

**原协议内涵**：收益权是私权利的回报层，指权利主体从数据资产的使用、衍生和扩展中获得经济收益的权利。收益分配遵循"谁贡献、谁受益"原则，由智能合约自动执行分润。

**城市公共数据适配**：
- **成本补偿权**：数据持有单位、运营单位对数据采集、存储、维护、脱敏等成本主张补偿的权利
- **公共价值回馈权**：数据经营收益在补偿成本后，以公共价值回馈形式反哺城市治理和公共服务的机制
- **审计监督权**：对收益来源、分配流向、使用效率进行审计监督的权利，确保"取之于民、用之于民"

**合约模块**：行为日志合约（收益追溯）+ 审计反馈层
- 行为日志合约：记录数据调用次数、经营收入、成本支出、分配明细
- 审计反馈层：定期生成收益审计报告，识别异常收益、不当分配、公共价值回馈不足

**行政监管映射**：收益权不强调个人化收益分配，而强调**收益透明和公共价值回馈**。审计机关对公共数据经营收益进行专项审计，财政部门监督收益使用方向，确保超额收益用于公共服务优化而非私人垄断。

## 三、与原协议的关键差异

| 维度 | ECHO 原协议（数字资产场景） | 本文适配（城市公共数据场景） |
|:---|:---|:---|
| **权利主体** | 个人创作者、数字资产持有者 | 政府部门、公共机构、受托管理方 |
| **权利性质** | 私权利，可转让、可交易 | 受托管理权，不可转让，受公共利益约束 |
| **争议解决** | 去中心化仲裁（社区投票、智能合约自动裁决） | 行政监管路径（合约校验—主管部门复核—审计监督） |
| **收益分配** | 市场化分润，创作者获得直接收益 | 成本补偿 + 公共价值回馈，收益反哺公共服务 |
| **技术架构** | 公链/联盟链，去中心化身份（DID） | 许可链/政务链，机构证书 + 权限白名单 |
| **代币机制** | 可能涉及代币激励 | 不涉及代币，规避政策风险 |
| **治理逻辑** | 社区自治 + 代码即法律 | 行政监管 + 技术辅助 + 人工复核 |

## 四、论文表述建议（第三部分完整段落）

> 本文对前期研究中形成的 ECHO 协议进行了场景化重构。原 ECHO 协议面向数字资产创作与流通场景，以"使用权—衍生权—扩展权—收益权"四私权利为核心架构，支撑创作者对数字成果的全生命周期治理：使用权界定访问与加工边界，衍生权保障再创作与成果归属，扩展权控制流通与经营范围，收益权实现"谁贡献、谁受益"的市场化分润。面向城市公共数据治理场景，本文将四私权利映射为"访问加工—衍生创新—共享经营—收益审计"四环节治理闭环，在保留原协议"确权—授权—使用—追责"核心逻辑的基础上，适配公共数据的受托管理属性、行政监管框架和公共利益导向。
>
> 具体而言，"访问加工"环节对应原协议的使用权，但将权利主体从个人创作者扩展为政府部门、公共机构和合规第三方，并通过授权控制合约与用途控制合约实现最小必要授权和越权自动拦截；"衍生创新"环节对应衍生权，但强调伦理审查、算法备案和成果血缘追溯，确保衍生成果可回溯至原始公共数据来源；"共享经营"环节对应扩展权，但将市场化扩展转为行政监管下的分级共享和授权运营，数据经营以公共价值回馈为前提；"收益审计"环节对应收益权，但将个人化收益分配转为成本补偿与公共价值回馈，审计机关和财政部门对收益流向进行专项监督。
>
> 与原协议的去中心化争议解决机制不同，本文设计的城市公共数据权利争议通过"智能合约自动校验—主管部门行政复核—审计机关专项审计"三级机制处理。智能合约承担第一层自动校验，对越权访问、超目的使用、超频调用和异常输出进行实时拦截；数据主管部门承担第二层行政裁定，对授权申请、权利冲突和共享审批进行人工复核；审计机关和财政监管保留为第三层收益监督，确保公共数据经营收益透明和公共价值回馈到位。这种设计既保留了 ECHO 协议的闭环治理逻辑，又符合我国行政监管体制、数据安全治理要求和公共利益导向。
