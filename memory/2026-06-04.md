## 📌 四权核心概念（最终对齐）

### 用权 = 消费的方法（谁能读/触发/调用）
- **粒度：** 细粒度，读/触发/调用分开控制
- **合约实现：** bitmask（READ_BIT | TRIGGER_BIT | INVOKE_BIT）
- **三档：** 公开/条件/私密（控制信息密度分布）

### 衍权 = 衍生的方法（谁能改编/翻译/混剪/续作）
- **方法集（7项定稿）：**
  1. 翻译
  2. 改编
  3. 混剪
  4. 续作
  5. 同义改写 / paraphrase
  6. 跨媒介转换（书→影→剧）
  7. 评论注解
- **Phase 2.1 暂缓：** 跨语种迁移/同人文/商业化重打包/抽象泛化

### 扩权 = 扩展的方法（让内容作为结构输入参与生长）
- **核心区分（云子提出，X7 确认）：**
  - 结构嵌入 = 扩权（自己的内容作为组件嵌入别人作品）
  - 势位引用 = 用权（引用别人的势位状态来触发自己的内容）
- **方法集：** 模板复用 / 子图组合 / 引用再锚 / 反向引证
- **数据增强归属：** 作为方法跨权复用（衍权=生成变体，扩权=结构输入）

### 益权 = 获益的方法（分账比例）
- **enforce 路径（最终定稿）：** 链下计算 + 链上不可绕过校验
  - 档位→比例映射存在链下（参数表，可调）
  - 链上只做校验：「应收额之和 = 100%」+「每个地址额 ≥ 0」
  - 缓释风险：参数表 hash 上链存证
  - Phase 2.1：预言机或 DAO 治理改参数

## 🎨 东方美学视角（云子提出，X7 采纳）

四权是技术概念，也是叙事概念，对应东方美学行为：

| 四权 | 行为 |
|:---|:---|
| 用权 | 观（读）/ 动（触发）/ 引（调用） |
| 衍权 | 化（翻译）/ 变（改编）/ 融（混剪）/ 续（续作）/ 转（跨媒介） |
| 扩权 | 嵌（结构嵌入）/ 锚（势位锚点）/ 连（级联触发）/ 网（引用再锚） |
| 益权 | 分（分账）/ 馈（回流）/ 积（累积） |

**年轮墨色体系（6档）：**
`#1A1A2E` → `#3D3D5C` → `#5C8A8A` → `#8C8C8C` → `#B5B5C8` → `#E8E4E1`
倍率：1.0x → 1.2x → 1.5x → 1.8x → 2.0x → 2.2x（tier6 修正为 2.2x，平滑相邻差值）

## 🔧 技术决策汇总

| 决策项 | 结论 |
|:---|:---|
| 用权粒度 | 细粒度（读/触发/调用分开，gas 差异 ~5k 可忽略） |
| 益权 enforce | 链下计算 + 链上不可绕过校验（参数表 hash 上链存证） |
| 六档倍率修正 | tier5 维持 2.0x，tier6 调至 2.2x，缩小跳差 |
| 衍权方法集 | 7项定稿（翻译/改编/混剪/续作/同义改写/跨媒介转换/评论注解） |
| 撤销机制 | 冷却期 + 引用锁定，原节点标记为「历史锚点」 |
| 撤销风险 | 已广播信息熵不可逆，大规模引用后撤销导致信息场畸变 |

## 📊 信息论视角贡献（X7）

**与 Shi-Graph 场论映射：**
- 熵 → 粒子分布无序度
- 互信息 → 节点间协同强度
- 信息瓶颈 → 合约层降维压缩

**Gas 优化策略：**
- 存储打包：Distribution struct 5 slot → 3 slot（节省 40%）
- 批量操作：单次 battle ≤50 张卡（gas ~2.5M）
- Merkle proof 方案：MVP 阶段不推荐，复杂度高

**数据增强信息论分类：**
- 生成变体 → 衍权（信息维度变换）
- 作为结构输入 → 扩权（信息拓扑聚合）
- 目的决定归属，非方法本身
